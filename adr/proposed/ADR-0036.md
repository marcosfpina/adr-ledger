## Context

The securellm-mcp ADR system currently has 34 ADRs (21 accepted, 11 proposed, 2 superseded). Agents create and query ADRs via MCP tools, but they are **not aware of open ADRs during their normal workflow**. This leads to:

1. **Decision drift** — agents make changes that contradict or duplicate open ADR proposals
2. **Staleness** — ADRs with expired `review_deadline` go unnoticed for weeks
3. **Manual triage burden** — architect must periodically review all open ADRs
4. **Missed closures** — completed work that fulfills an ADR is not automatically linked

The Pre-Action Interceptor (`src/reasoning/proactive/pre-action-interceptor.ts`) already validates git status and build environment before tool calls. The ContextManager already enriches context with project state and knowledge base queries. **The infrastructure for ADR-awareness exists — we just need to wire it.**

## Decision

Extend securellm-mcp with three components that make agents ADR-aware automatically:

### Phase 1: ADR Context Injector

**Location:** `src/reasoning/proactive/pre-action-interceptor.ts`

Before any file-modifying tool call, extract keywords from the tool arguments and query `adr_search`. If relevant proposed ADRs exist, inject a brief context note into the enriched response:

```typescript
// In PreActionInterceptor.intercept():
async enrichWithADRContext(toolName: string, args: any): Promise<string | null> {
  // Only for file-modifying tools
  if (!FILE_MODIFYING_TOOLS.has(toolName)) return null;

  // Extract keywords from file paths and content
  const keywords = this.extractKeywords(args);

  // Search proposed ADRs (cached, TTL 5min)
  const relevant = await this.adrBackend.searchProposed(keywords, { limit: 2 });

  if (relevant.length === 0) return null;

  return relevant.map(adr =>
    `[ADR-${adr.id}] ${adr.title} (${adr.status}) — ${adr.summary}`
  ).join('\n');
}
```

**Caching:** ADR index cached in memory with 5-minute TTL. Cost: ~5ms per lookup after first load.

### Phase 2: ADR Staleness Monitor

**Location:** new `src/middleware/adr-hygiene.ts`

A lightweight middleware that runs every N tool calls (configurable, default 50):

```typescript
class ADRHygieneMiddleware {
  private callCount = 0;
  private readonly CHECK_INTERVAL = 50;

  async check(): Promise<ADRHygieneReport> {
    const proposed = await this.backend.list({ status: 'proposed' });

    return {
      stale: proposed.filter(adr =>
        daysSince(adr.date) > 30 && !adr.recentActivity
      ),
      expired_deadline: proposed.filter(adr =>
        adr.governance.review_deadline && isPast(adr.governance.review_deadline)
      ),
      conflicting: this.detectConflicts(proposed),
      total_open: proposed.length,
    };
  }
}
```

When issues are found, surfaces them as a tool result note: `"ADR Hygiene: 3 stale, 2 expired deadlines. Run adr_list --status=proposed to review."`

### Phase 3: ADR Auto-Lifecycle

**Location:** `src/tools/adr/storage/git-backend.ts`

After an `adr_accept` or when work is committed that references an ADR:

```typescript
async suggestLifecycleChange(completedWork: string): Promise<Suggestion[]> {
  const proposed = await this.list({ status: 'proposed' });
  const suggestions: Suggestion[] = [];

  for (const adr of proposed) {
    // Check if all tasks are done
    const tasks = this.parseTasks(adr.content);
    if (tasks.length > 0 && tasks.every(t => t.done)) {
      suggestions.push({
        adr: adr.id,
        action: 'accept',
        reason: `All ${tasks.length} tasks completed`,
      });
    }

    // Check if superseded by newer ADR
    const newer = proposed.filter(other =>
      other.id !== adr.id && this.topicOverlap(adr, other) > 0.7
    );
    if (newer.length > 0) {
      suggestions.push({
        adr: adr.id,
        action: 'supersede',
        reason: `Overlaps with ${newer.map(n => n.id).join(', ')}`,
      });
    }
  }

  return suggestions;
}
```
