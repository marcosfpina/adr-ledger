## Context

O `securellm-mcp` expoe 21 tools ADR (13 read + 8 write) via MCP protocol. Essas tools estao corretamente registradas no codigo TypeScript (`src/tools/adr/index.ts`) e compiladas no build JS. Porem, o Claude Code so reconhece 6 delas ao conectar.

**Problema raiz**: inconsistencia entre como o MCP server e invocado:

| Projeto | `command` | `args` | Tools reconhecidas |
|---------|-----------|--------|-------------------|
| securellm-mcp | `node` | `[build/src/index.js]` | 63 (todas) |
| adr-ledger | `securellm-mcp` | `[]` | ~40 (parcial) |

O wrapper Nix `securellm-mcp` (instalado via `nix profile` ou `environment.systemPackages`) pode estar com uma versao desatualizada do build, ou com env vars diferentes que afetam o registro de tools.

Alem disso, o campo `capabilities.tools.count` no `.mcp.json` e mantido manualmente e diverge do real.

## Decision

1. **Padronizar invocacao**: todos os `.mcp.json` devem usar `command: node` + `args: [path/to/build/src/index.js]` durante desenvolvimento, e o wrapper Nix apenas em producao (apos rebuild do sistema)

2. **Gerar manifest automaticamente**: adicionar `npm run build:manifest` que executa `tools/list` contra o server e gera o bloco `capabilities.tools` com contagem e nomes reais

3. **CI gate**: adicionar check que valida `tools/list` count >= expected count, detectando regressoes

4. **Sincronizar wrapper Nix**: garantir que `nixos-rebuild` ou `nix profile upgrade` atualiza o wrapper apos cada `npm run build`

## Implementation
