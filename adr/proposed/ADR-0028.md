## Contexto

### Por que Vertex AI existe no CEREBRO?

Em 11 de janeiro de 2026, **ADR-0007** estabeleceu a estrategia **Local-First AI** para PHANTOM e CEREBRO, com embeddings via `sentence-transformers/all-MiniLM-L6-v2` (384-dim, custo $0).

**4 dias depois** (15 jan, commit `63e71f9`), o `embedding_server.py` foi adicionado ao CEREBRO usando **Vertex AI text-embedding-004** (768-dim, cloud). A motivacao:

> **$6k de creditos GCP disponiveis** — decisao oportunista de aproveitar qualidade SOTA sem custo imediato.

Nenhum ADR foi criado na epoca documentando essa excecao ao ADR-0007. O comentario no `start_embedding_server.sh` linha 21 confirma a intencao:
```
💰 Ready to burn those $6k credits! 🔥
```

### O que deu errado

1. **Crash loop #1** (1630+ restarts): `WorkingDirectory` apontava para `/home/kernelcore/arch/cerebro` — path stale apos mover repo para `/home/kernelcore/master/cerebro`
2. **Crash loop #2** (18,192+ linhas de log): `ModuleNotFoundError: No module named 'vertexai'` — o `flake.nix` nao inclui vertexai, e o shellHook do Poetry nao executa no contexto do systemd ExecStart
3. **Nunca conectado ao consumidor**: securellm-mcp aponta para `localhost:8081` (llama.cpp), nao `8001` (cerebro)
4. **Dimensoes incompativeis**: Vertex AI retorna 768-dim, fallback do securellm-mcp usa 384-dim

### Consumidor: securellm-mcp Semantic Cache

O `SemanticCache` em `securellm-mcp/src/middleware/semantic-cache.ts` usa embeddings para deduplicar tool calls MCP. Fluxo:

```
Tool call recebida → generateEmbedding(query_text)
    → fetch(LLAMA_CPP_URL/embedding, {content: text})
    → cosineSimilarity(query, cached_entries)
    → Se similarity > 0.85 → cache hit (evita chamada ao LLM)
```

Config atual (`src/types/semantic-cache.ts:88`):
```typescript
llamaCppUrl: process.env.LLAMA_CPP_URL || 'http://localhost:8081'
```

Fallback quando embedding server indisponivel (`semantic-cache.ts:200-228`):
```typescript
// Char frequency hash → Float32Array(384) — rapido mas impreciso
private fallbackEmbedding(text: string): Float32Array { ... }
```

### API Compativel (Anti-Lock-In Natural)

O cerebro-embeddings ja foi desenhado como **drop-in replacement** do llama.cpp:

| Aspecto | llama.cpp (8081) | cerebro-embeddings (8001) |
|---------|------------------|---------------------------|
| Endpoint | `POST /embedding` | `POST /embedding` |
| Request | `{ content: string }` | `{ content: string }` |
| Response | `{ embedding: float[] }` | `{ embedding: float[], dimension, model, cost }` |
| Dimensao | Depende do modelo | 768 (text-embedding-004) |

A troca de provider requer **apenas mudar a URL/porta**. Zero mudanca de codigo.

## Decisao

**Manter cerebro-embeddings com Vertex AI como solucao transitoria** (ate ~maio 2026), com plano de migracao para local-first documentado e calendarizado.

### Calculo de Risco: Creditos GCP

| Metrica | Valor |
|---------|-------|
| Creditos iniciais | $6,000 |
| Custo por embedding | ~$0.00001 / 1k chars |
| Queries estimadas/dia | ~500 (tool calls MCP) |
| Chars medios/query | ~100 chars |
| Custo diario estimado | ~$0.0005 |
| Custo mensal estimado | ~$0.015 |
| **Creditos duram** | **Muito alem de 3 meses** (custo negligivel) |
| **Deadline real** | **Expiracao dos creditos (~maio 2026)**, nao consumo |

> O risco NAO e gastar os creditos — o custo de embedding e negligivel. O risco e a **expiracao temporal** dos creditos e a **dependencia de infraestrutura GCP** (auth, rede, API stability).

### Anti-Lock-In: Abstraction Layer

O design atual ja previne lock-in naturalmente:

1. **API identica**: `POST /embedding` com `{content: str}` → `{embedding: float[]}` — funciona com qualquer backend
2. **Config por env var**: `LLAMA_CPP_URL` — trocar de `localhost:8001` para `localhost:8081` e suficiente
3. **Graceful degradation**: fallback char-frequency ja funciona quando server indisponivel
4. **Cache efemero**: TTL de 1h, limpar na migracao nao causa perda de dados

**Nenhuma camada de abstracao adicional e necessaria.** A arquitetura existente ja permite swap sem codigo.

#### 1.2 Migrar service para home-manager

Criar: `/etc/nixos/hosts/kernelcore/home/cerebro-embeddings.nix`

```nix
{ config, pkgs, ... }:
{
  systemd.user.services.cerebro-embeddings = {
    Unit = {
      Description = "CEREBRO Embedding Server (Vertex AI)";
      After = [ "network-online.target" ];
    };
    Service = {
      Type = "simple";
      WorkingDirectory = "/home/kernelcore/master/cerebro";
      ExecStart = "${pkgs.bash}/bin/bash -c 'cd /home/kernelcore/master/cerebro && nix develop --command python -m uvicorn phantom.core.embedding_server:app --host 127.0.0.1 --port 8001'";
      EnvironmentFile = "%h/.config/sops/gcp-credentials.env";
      Restart = "on-failure";
      RestartSec = "10s";
      MemoryMax = "2G";
      CPUQuota = "200%";
    };
    Install = {
      WantedBy = [ "default.target" ];
    };
  };
}
```

Adicionar ao env do securellm-mcp:

