## Context

NEUTRON é o "núcleo atômico" da stack - a camada de infraestrutura que sustenta tudo. O nome vem da física: assim como neutrons mantêm o núcleo estável, NEUTRON mantém a infraestrutura estável e previsível.

Sem infraestrutura confiável, os outros sistemas (CEREBRO, SPECTRE, PHANTOM) não têm onde rodar. NEUTRON é a fundação sobre a qual tudo é construído.

### ADR Compliance Enforcement

NEUTRON lê ADRs e enforça automaticamente:

```nix
# modules/governance/adr-compliance.nix
{ config, lib, pkgs, ... }:

let
  adrLedger = builtins.fromJSON (builtins.readFile /path/to/knowledge_base.json);
  
  # Extrai regras de compliance de ADRs aceitas
  complianceRules = lib.filter 
    (adr: lib.hasAttr "SECURITY" adr.governance.compliance_tags)
    adrLedger.decisions;
in {
  assertions = map (rule: {
    assertion = /* check compliance */;
    message = "ADR ${rule.id} compliance violation: ${rule.title}";
  }) complianceRules;
  
  # Bloqueia rebuild se violar ADRs de segurança
  system.activationScripts.adrCompliance = ''
    echo "Checking ADR compliance..."
    # ... validation logic
  '';
}
```

## Rationale

### Por que NixOS?

| Propriedade | Benefício |
|-------------|-----------|
| **Declarativo** | `configuration.nix` = estado completo do sistema |
| **Reproduzível** | Mesmo input → mesmo output, sempre |
| **Atômico** | Rollback instantâneo para gerações anteriores |
| **Isolado** | Cada pacote em seu próprio path |
| **Versionado** | Lockfiles garantem versões exatas |

### Security Hardening

```nix
# Exemplo de hardening aplicado
security = {
  # Kernel hardening
  protectKernelImage = true;
  lockKernelModules = true;
  
  # AppArmor
  apparmor.enable = true;
  apparmor.killUnconfinedConfinables = true;
  
  # Audit
  auditd.enable = true;
  audit.rules = [
    "-w /etc/passwd -p wa -k identity"
    "-w /etc/shadow -p wa -k identity"
    # ... LGPD-required auditing
  ];
};
```

## Consequences

### Operational Excellence

Com NEUTRON baseado em NixOS:

1. **Disaster Recovery**: `nixos-rebuild switch` restaura sistema inteiro
2. **Debugging**: `nixos-option` mostra exatamente o que está configurado
3. **Updates**: `nix flake update` + rebuild atômico
4. **Audit**: Histórico completo em `git log` do flake

### Métricas de Sucesso

- **MTTR** (Mean Time to Recovery): <5 minutos via rollback
- **Configuration Drift**: Zero (by design)
- **Compliance Coverage**: 100% das ADRs de security enforced
- **Uptime**: >99.9%
- **Security Posture**: CIS Benchmark Level 2

### Trade-offs Aceitos

1. **Learning Curve**: NixOS requer investimento significativo
   - Mitigação: Documentação extensiva, módulos bem comentados

2. **Ecosystem Size**: Menos pacotes que Ubuntu/Fedora
   - Mitigação: Containers/VMs para software não-nixificado

3. **Build Times**: Compilação pode ser lenta
   - Mitigação: Binary cache, Cachix
