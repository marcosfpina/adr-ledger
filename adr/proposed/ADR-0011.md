## Context

O ecossistema de projetos `low-level` atingiu um ponto onde múltiplos componentes (Phantom, SecureLLM, Spectre, ADR-Ledger) estão funcionais isoladamente, mas carecem de uma "liga" que demonstre seu valor combinado. 

Simultaneamente, o projeto **Cerebro** (destinado a ser o cérebro do sistema) apresenta sinais de fragmentação durante sua migração (conflitos de versão de Python e scripts obsoletos).

## Decision

Decidimos implementar o MVP **"The Self-Aware Architect"**. 

Este MVP consiste em transformar o `adr-ledger` (estático) em um assistente de arquitetura dinâmico (RAG) usando o `phantom` como motor de processamento e o `securellm-mcp` como interface para o desenvolvedor.

### Arquitetura da "Liga"

1. **Source**: `adr-ledger` fornece os arquivos Markdown/JSON.
2. **Processing**: `phantom` processa os documentos, gera embeddings e armazena em um índice FAISS local.
3. **Interface**: `securellm-mcp` ganha uma nova `tool` que consulta esse índice e retorna respostas contextualizadas para a LLM (Claude/Cline).

## Consequences

Esta decisão permite que o sistema comece a "entender a si mesmo" (Self-Awareness), respondendo perguntas sobre suas próprias decisões de design. 

A agilidade é priorizada, aceitando-se o risco de refatoração futura em prol de uma prova de valor imediata para o portfólio.
