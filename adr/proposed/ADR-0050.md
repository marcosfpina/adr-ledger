## Context

O workspace `/home/kernelcore/master` é um monorepo Nix-first com múltiplos projetos interdependentes. Após exploração completa do ecossistema, 4 streams de entrega foram identificados com gaps concretos a fechar — não reescritas.

**Spectre** é o backbone de eventos NATS, já production-ready (Phase 4 completa). Os demais projetos dependem dele para comunicação assíncrona, mas faltam os event types da camada de rede e do domínio phantom-stack.

**phantom-soc** (control-plane Rust + GTK4) tem um bug crítico no scheduler que impede qualquer task de ser processada — loop infinito de sleep sem dequeue.

**owasaka** (sensor Go de rede) faz descoberta ARP e queries DNS mas não publica nada no NATS.

**phantom** (ML platform Python/FastAPI) tem 7 endpoints stub que bloqueiam o Cortex Desktop (Tauri + Svelte).

**neoland** (Rust TUI) perde todos os vetores a cada restart por usar store in-memory.

## Decision

Executar o roadmap em ordem de dependência, um projeto por vez, com mudanças cirúrgicas e verificação de build/testes após cada task:

```
A1 (spectre events) → A2 (scheduler fix) → A3 (owasaka NATS)
→ A4 (data-plane subscribe) → A5 (GTK4 wire)
→ B1 (phantom endpoints) → B2 (cortex wire) → B3 (manifest)
→ C1 (neoland vectors) → C2+C3 (tests + deps)
```

## Rationale

### Drivers

- Spectre sem `network.*` event types bloqueia owasaka de publicar eventos tipados
- Scheduler com infinite sleep loop bloqueia toda a pipeline phantom-soc
- 7 endpoints stub no phantom bloqueiam o frontend Cortex Desktop
- Vetores in-memory em neoland tornam o produto inviável para uso real

### Trade-offs

- Trabalho sequencial (não paralelo) por stream — aceita velocidade menor em troca de integração mais segura

## Consequences

### Positive

- phantom-soc scheduler funcional desbloqueia toda a pipeline de inteligência de segurança
- owasaka como sensor de rede integrado ao barramento NATS via Spectre
- phantom API completa desbloqueia Cortex Desktop end-to-end
- neoland com persistência SQLite viável para uso em produção

### Negative

- Novas dependências: `nats.go` em owasaka, `rusqlite` + `sqlite-vec` em neoland

### Risks

| Risk | Probability | Impact | Mitigation |
|------|-------------|--------|------------|
| Scheduler fix requer refactor de ownership | medium | medium | Mover receiver para dentro do Scheduler |
| sqlite-vec não disponível no nixpkgs | low | high | Usar bundled extension ou nix overlay |
