## Context

The securellm-mcp codebase (121 files, ~33K LOC, 80+ MCP tools) has two files that far exceed reasonable module size:

| File | LOC | Concern |
|------|-----|---------|
| `src/index.ts` | **2632** | Tool listing, handler dispatch, resource handlers, lifecycle, init, shutdown |
| `src/types/infrastructure.ts` | **2020** | K8s types, firewall rules, volume config, deployment validation, CIDR parsing |

Secondary candidates (800-1000 LOC):
- `src/knowledge/database.ts` (989) - CRUD + FTS5 + context + compaction triggers
- `src/tools/web-search.ts` (977) - 6 search providers in one file
- `src/types/ssh-advanced.ts` (807) - tunnels, jump hosts, session management types

These large files cause: slow IDE navigation, frequent merge conflicts on `index.ts`, difficulty testing individual subsystems, and long TypeScript incremental recompilation.

## Decision

Split files exceeding 1500 LOC into domain-focused modules:

**`src/index.ts` (2632 LOC) → 4 files:**
- `src/registry/tool-registry.ts` - tool listing (ListToolsRequestSchema handler)
- `src/registry/tool-dispatcher.ts` - handler dispatch (CallToolRequestSchema handler)
- `src/registry/resource-handlers.ts` - resource listing and reading
- `src/index.ts` - thin entry point: constructor, init, wire registries, start transport

**`src/types/infrastructure.ts` (2020 LOC) → 4 files:**
- `src/types/k8s.ts` - Kubernetes deployment, service, resource types + validation
- `src/types/firewall.ts` - Firewall rules, CIDR validation
- `src/types/volumes.ts` - Volume config, storage types
- `src/types/infrastructure.ts` - re-exports for backward compatibility (barrel file)
