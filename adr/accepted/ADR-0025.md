## Context

The 10-phase Phantom production readiness audit (ADR-0024) completed on 2026-02-04.
Phase 2 archived `cortex_chunker.py` and `cortex_embeddings.py` as dead code because a
grep over `src/` showed no imports — it missed `tests/unit/` which actively imports both.
Phases 3 and 10 added/removed deps in `pyproject.toml` without mirroring the changes in
`flake.nix` `pythonEnv`, so `prometheus-client`, `structlog`, `httpx`, `typer`, and `faiss`
were all absent from the Nix dev shell. Several interfaces declared in `__init__.py`
(`LatencyOptimizer`, `SpectreAnalyzer.analyze`, `SanitizationEngine.scan_sensitive_patterns`)
had no implementation. The `/rag/query` endpoint was registered as POST but every consumer
used GET.

Running `pytest` after the audit produced 5 collection errors and, once those were fixed,
12 runtime failures. Zero tests passed cleanly end-to-end.

## Decision

Restore the archived modules, implement the missing interfaces directly in the existing
source files (no new abstractions), sync `flake.nix` with `pyproject.toml`, and fix the
handful of test/route mismatches. Do not delete tests to make them pass — the modules are
real and required by the RAG pipeline.

## Rationale

### Drivers

- Phase 2 dead-code grep only searched `src/`; `tests/` was invisible to it.
- `flake.nix` is the runtime source of truth inside `nix develop`; `pyproject.toml` alone is not enough.
- Interfaces declared in `__all__` must exist — lazy `__getattr__` will blow up at first access otherwise.

### Alternatives Considered

#### Option A: Delete the failing tests
- **Why rejected:** `cortex_chunker` and `cortex_embeddings` are real, complete modules used
  by the embedding pipeline. Removing their tests would leave the RAG layer untested.

### Trade-offs

- `cortex_chunker` and `cortex_embeddings` have some surface overlap with `vectors.py`, but
  they serve distinct roles (chunking strategy vs. vector store abstraction). Consolidation is
  future work, not a blocker.

## Consequences

### Positive

- **88/88 tests pass, 0 skipped.** Full coverage of imports, chunking, embeddings, FAISS,
  sentiment, spectre, sanitization, classification, and the FastAPI endpoints.
- `flake.nix` and `pyproject.toml` are in sync. The invariant is documented here so future
  phases cannot silently diverge.
- `SpectreAnalyzer.analyze(text)` enables in-process NLP without writing temp files — used
  by the Judge API sentiment path.
- `SanitizationEngine.scan_sensitive_patterns(text)` is now callable from any service that
  needs PII detection on already-loaded strings.

### Negative

- Core CLI entry-points (`phantom-scan`, `phantom-hash`, `phantom-api`) are still only
  exercised manually. Automated tests for those are pending.

### Risks

| Risk | Probability | Impact | Mitigation |
|------|-------------|--------|------------|
| Future dep changes silently break flake | medium | high | This ADR documents the invariant; CI lint step should diff both files |
