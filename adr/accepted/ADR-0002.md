## Context

CEREBRO é o "cérebro" da stack - a camada que transforma dados brutos em knowledge acionável. Sem ele, cada sistema opera em silo, sem contexto compartilhado.

O problema: LLMs têm context window limitado. Não podemos injetar todo o knowledge base em cada prompt. RAG resolve isso via retrieval semântico.

## Decision

CEREBRO implementa **Retrieval Augmented Generation** com:

```
┌─────────────────────────────────────────────────────────────┐
│                      CEREBRO Architecture                    │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐              │
│  │  Ingest  │───▶│  Index   │───▶│  Store   │              │
│  │ Pipeline │    │ (Chunk)  │    │ (Vector) │              │
│  └──────────┘    └──────────┘    └──────────┘              │
│       ▲                               │                     │
│       │                               ▼                     │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐              │
│  │   ADR    │    │  Query   │◀───│ Retrieve │              │
│  │  Ledger  │    │   API    │    │ (Top-K)  │              │
│  └──────────┘    └──────────┘    └──────────┘              │
│                       │                                     │
│                       ▼                                     │
│              ┌──────────────┐                               │
│              │   Generate   │                               │
│              │ (LLM + Ctx)  │                               │
│              └──────────────┘                               │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### Componentes

1. **Ingest Pipeline**: Consome ADRs, docs, código
2. **Chunking**: Estratégia semântica (não fixed-size)
3. **Embedding**: text-embedding-3-large (1536 dims)
4. **Vector Store**: pgvector (PostgreSQL) ou Qdrant
5. **Query API**: REST + gRPC para outros sistemas
6. **Generation**: LLM com context injetado

### Auto-Ingestão de ADRs

Quando `adr sync` executa:
```bash
# 1. Parser gera knowledge_base.json
# 2. CEREBRO detecta mudança (inotify/polling)
# 3. Chunks novos/modificados são re-indexed
# 4. Cache invalidado para queries afetadas
```

## Rationale

### Por que RAG vs Fine-tuning?

| Aspecto | RAG | Fine-tuning |
|---------|-----|-------------|
| **Update speed** | Minutos | Horas/dias |
| **Cost per update** | ~$0.01 | ~$50-500 |
| **Auditability** | Alta (sources) | Baixa |
| **Scalability** | Linear | Logarítmica |
| **Accuracy control** | Threshold tuning | Retraining |

### Proactive Learning

CEREBRO não é apenas retrieval passivo. Ele:

1. **Detecta gaps**: "Usuários perguntam X mas não temos ADR para isso"
2. **Sugere ADRs**: Propõe novas decisões baseado em patterns
3. **Cross-references**: Identifica conflitos entre decisões

- **Index Freshness**: <5min após ADR sync
