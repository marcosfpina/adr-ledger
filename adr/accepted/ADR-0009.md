## Context

NEUTRON orchestrates ML workflows for hyperparameter optimization, model training, and experimentation.

### Problem: Manual Hyperparameter Tuning

**Traditional Approach**:
```python
# Engineer manually tries hyperparameters
for lr in [0.001, 0.01, 0.1]:
    for batch_size in [32, 64, 128]:
        model = train_model(lr=lr, batch_size=batch_size)
        accuracy = evaluate(model)
        print(f"lr={lr}, batch_size={batch_size}, accuracy={accuracy}")
        # Manually log to CSV
```

**Problems**:
- **Time-consuming**: Engineer babysits training runs
- **No fault tolerance**: Crash loses all progress
- **Suboptimal**: No intelligent search strategy
- **Poor tracking**: CSV files, manual notes
- **Wasted GPU**: Redundant model loads per trial

## Decision

Adopt **Temporal + Ray + MLflow** stack for adaptive ML orchestration.

### Adaptive Hyperparameter Optimization

**4 Strategies** (src/neutron/optimizer.py):

1. **GRID**: Exhaustive systematic search (small discrete spaces)
2. **RANDOM**: Fast exploration, good baseline (always solid)
3. **BAYESIAN**: Sample-efficient, models uncertainty (expensive evals)
4. **EVOLUTIONARY**: Creative discovery, finds unexpected regions

**Adaptive Logic**:
```python
# Phase 1: Exploration (GRID)
trials_phase1 = run_trials(strategy=GRID, n=20)
convergence = analyze_convergence(trials_phase1)

if convergence < 0.8:  # Not converged, explore more
    # Phase 2: Random exploration
    trials_phase2 = run_trials(strategy=RANDOM, n=20)
else:
    # Phase 2: Exploitation (BAYESIAN)
    trials_phase2 = run_trials(strategy=BAYESIAN, n=10)

# Phase 3: Final refinement (EVOLUTIONARY or BAYESIAN)
best_region = identify_promising_region(trials_phase1 + trials_phase2)
trials_phase3 = run_trials(strategy=BAYESIAN, n=10, focus=best_region)
```

**Why Adaptive?**
- Fixed GRID: Wastes trials on unimportant dimensions
- Fixed RANDOM: May miss optima in complex spaces
- **Adaptive**: Starts broad (GRID/RANDOM), narrows to promising regions (BAYESIAN)

### Temporal Durable Execution

**Problem** (Airflow):
```python
# Airflow DAG
def train_workflow():
    for i in range(50):
        train_model(trial=i)  # Trial 25: Server crashes
    # On restart: Start from trial 0 again (lost 25 trials)
```

**Solution** (Temporal):
```python
# Temporal workflow
@workflow.defn
class AdaptiveMLPipelineWorkflow:
    @workflow.run
    async def run(self, config: PipelineConfig):
        for i in range(50):
            result = await workflow.execute_activity(
                train_model_activity,
                args=(i,),
                start_to_close_timeout=timedelta(minutes=10)
            )
            # Trial 25: Server crashes
            # On restart: Temporal replays from event log
            # Resumes at trial 26 (deterministic replay)
```

**Benefits**:
- Crash recovery: Resume from last completed activity
- Deterministic: Same inputs → same outputs (reproducibility)
- Long-running: Workflows can run for days/weeks

### Ray Stateful Actors

**Problem** (Celery):
```python
# Celery task (stateless)
@task
def train_trial(hyperparams):
    model = load_model()  # Loads from disk every trial (slow)
    result = train(model, hyperparams)
    return result
```

**Solution** (Ray):
```python
# Ray actor (stateful)
@ray.remote(num_gpus=1)
class TrainerPool:
    def __init__(self):
        self.model = load_model()  # Load once, cache in GPU VRAM

    def train_trial(self, hyperparams):
        # Model already in VRAM, instant access
        result = train(self.model, hyperparams)
        return result

# Spawn 4 actors (reuse across trials)
actors = [TrainerPool.remote() for _ in range(4)]
results = ray.get([actor.train_trial.remote(hp) for actor in actors])
```

**Benefits**:
- **Speed**: 1 model load → 50 trials (vs 50 loads)
- **GPU efficiency**: Model stays resident in VRAM
- **Parallelism**: 4 actors = 4× throughput

## Rationale

### Comparison

| Feature                | Airflow         | Kubeflow        | Temporal + Ray |
|------------------------|-----------------|-----------------|----------------|
| **Durable Execution**  | ❌ No            | ⚠️ Partial       | ✅ Full         |
| **Adaptive Workflows** | ❌ Static DAGs   | ❌ Static        | ✅ Dynamic      |
| **GPU Scheduling**     | ⚠️ Manual        | ✅ Kubernetes    | ✅ Ray actors   |
| **State Management**   | ⚠️ External DB   | ⚠️ K8s ConfigMap | ✅ Built-in     |
| **Learning Curve**     | ✅ Low           | ⚠️ Medium        | ⚠️ High         |
| **Fault Tolerance**    | ⚠️ Task retry    | ✅ Pod restart   | ✅ Workflow resume |

**Winner**: Temporal + Ray for durable execution + adaptive workflows
