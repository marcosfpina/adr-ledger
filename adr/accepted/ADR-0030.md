## Context

Cerebro suffers from an **identity crisis** with 3 conflicting framings:
1. **README.md** — "Enterprise knowledge extraction platform" (correct)
2. **docs/EXECUTIVE_SUMMARY.md** — "Phoenix Credit Burn ROI Strategy" (personal/career)
3. **README_PORTFOLIO.md** — "Production-grade code intelligence" (portfolio)

Additional issues: inconsistent naming (Phantom vs Cerebro), Portuguese strings in CLI/TUI,
"burn" language dominating product identity, 12 proposed ADRs pending triage, and 16 loose
markdown files in the project root.

## Decision

Reposition Cerebro as an **enterprise catalyst**:

1. **Naming**: Cerebro = product name, phantom = internal Python module (kept for backward compat)
2. **Language**: All user-facing strings in English; Portuguese removed from CLI/TUI
3. **Framing**: GCP credit consumption reframed as "batch execution" — a feature, not the identity
4. **Documentation**: 16 root markdowns moved to `docs/` subdirectories (nothing deleted)
5. **Hardcoded values**: GCP project IDs replaced with env vars / placeholders
6. **ADR triage**: ADR-0027 accepted; this ADR (0030) supersedes ADR-0017 and ADR-0024

This ADR does NOT:
- Rename `src/phantom/` directory (high risk, zero user value)
- Delete any files (only moves)
- Rewrite command logic (only reframes labels/docstrings)
- Remove the `gcp burn` subcommand (stays as a feature)

## Rationale

### Drivers

- Enterprise positioning requires consistent English-language professional branding
- "Burn" as product identity alienates enterprise adoption
- Portuguese UI limits international contribution and adoption
- Cluttered project root signals lack of organization

### Alternatives Considered

#### Option A: Full package rename (phantom → cerebro)
- **Pros:** Perfect consistency
- **Cons:** Breaks all imports, test fixtures, CI pipelines
- **Why rejected:** High risk for zero user-facing value. The `src/phantom/` directory is internal.

#### Option B: Delete burn-related commands entirely
- **Pros:** Clean enterprise image
- **Cons:** Loses useful GCP credit management functionality
- **Why rejected:** Credit management is a legitimate enterprise feature when properly framed.

### Trade-offs

- Internal module name `phantom` remains — acceptable since users interact via `cerebro` CLI
- Both `phantom` and `cerebro` CLI entrypoints maintained for backward compatibility

## Consequences

### Positive

- Consistent "Cerebro" branding across all user-facing surfaces
- English-only CLI enables international users and contributors
- GCP credit management positioned as enterprise feature
- Clean project root with organized documentation hierarchy
- ADR backlog triaged and actionable

### Negative

- Internal code still uses `phantom` module name (cosmetic inconsistency)
- Existing documentation URLs may change (mitigated by git history)

### Risks

| Risk | Probability | Impact | Mitigation |
|------|-------------|--------|------------|
| Test breakage from string changes | low | medium | Full test suite run after changes |
| Documentation links broken | low | low | Git history preserves old paths |
