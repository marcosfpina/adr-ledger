## Contexto

O servico `cerebro-embeddings` (Vertex AI, port 8001) entra em **sunset** em 2026-05-15 (ADR-0028). O securellm-mcp semantic cache precisa de um provider de embeddings real — atualmente roda 100% no fallback de char-frequency hash, que nao captura semantica.

Este ADR propoe um **novo projeto** focado em reranking e embeddings hibridos, com arquitetura local-first e cloud como fallback opcional.

### Problema

```
Hoje:
  securellm-mcp tool call → generateEmbedding()
      → fetch(localhost:8081/embedding)  ← llama.cpp (nao configurado para embeddings)
      → FALHA
      → fallbackEmbedding() ← char frequency hash (impreciso)
      → cosine similarity com cache
      → hit rate provavelmente baixo (embeddings nao sao semanticas)
```

### Solucao

```
Depois (ADR-0035):
  securellm-mcp tool call → HybridEmbeddingProvider.embed()
      → 1. Local fast (sentence-transformers/all-MiniLM-L6-v2, <5ms)
      → 2. Se incerteza: Local accurate (CrossEncoder ms-marco-deberta, ~50ms)
      → 3. Se falha local: Vertex AI fallback (enquanto creditos, ~100ms)
      → 4. Se tudo falha: char frequency hash (ultimo recurso)
      → cosine similarity com cache
      → hit rate significativamente melhor
```

## Decisao

Criar um **componente de embedding/reranking hibrido** com cadeia de fallback inteligente:

### Arquitetura

```

│              HYBRID RERANKER / EMBEDDER              │
├─────────────────────────────────────────────────────┤
```

### Cadeia de Fallback (Embedding)

```python
async def embed(self, text: str) -> list[float]:
    """Gera embedding com fallback chain."""

    # 1. Local fast — sentence-transformers (primario)
    try:
        return self.local_model.encode(text).tolist()
    except Exception:
        pass

    # 2. Cloud — Vertex AI (enquanto creditos vivos, opcional)
    if self.vertex_enabled:
        try:
            return await self._vertex_embed(text)
        except Exception:
            pass

    # 3. Hash fallback — ultimo recurso
    return self._char_frequency_hash(text)
```

### Cadeia de Fallback (Reranking)

```python
async def rerank(self, query: str, docs: list[str], mode='auto') -> list[dict]:
    """Rerank com selecao inteligente de modelo."""

    if mode == 'auto':
        # Fast pass primeiro
        scores = self.fast_reranker.predict([[query, d] for d in docs])

        # Se confianca alta, retorna
        if max(scores) > 0.8:
            return self._format(docs, scores)

        # Incerteza → modelo mais preciso
        scores = self.accurate_reranker.predict([[query, d] for d in docs])
        return self._format(docs, scores)

    elif mode == 'cloud' and self.vertex_enabled:
        return await self._vertex_rerank(query, docs)
```

### Integracao com securellm-mcp

Opcao A — **Micro-server HTTP** (mesma API do llama.cpp):
```
HybridReranker → FastAPI → POST /embedding → {embedding: float[]}
securellm-mcp → LLAMA_CPP_URL=http://localhost:PORT → funciona sem mudanca
```

Opcao B — **Node.js native** (via onnxruntime-node):
```
securellm-mcp → import { HybridEmbedder } from './hybrid-embedder'
              → in-process, zero HTTP overhead
```

Opcao A e mais simples e compativel com a arquitetura atual. Opcao B e mais performante mas requer rewrite do semantic cache.

**Recomendacao**: Comecar com **Opcao A** (micro-server), migrar para B se performance justificar.

## Modelos Recomendados

### Para Embeddings (semantic cache)

| Modelo | Dim | Tamanho | Latencia | Qualidade | Disponivel |
|--------|-----|---------|----------|-----------|------------|
| `all-MiniLM-L6-v2` | 384 | 80MB | <5ms | Boa | Ja no Nix devShell |
| `nomic-embed-text-v1.5` | 768 | 274MB | <10ms | Muito boa | GGUF p/ llama.cpp |
| `bge-m3` | 1024 | 1.1GB | ~20ms | Excelente | HuggingFace |

**Recomendacao**: `all-MiniLM-L6-v2` — ja disponivel, qualidade suficiente para tool call dedup, menor footprint.

### Para Reranking (RAG pipeline)

| Modelo | Tamanho | Latencia/par | Qualidade | Use case |
|--------|---------|-------------|-----------|----------|
| `ms-marco-MiniLM-L-6-v2` | 80MB | ~10ms | Boa | Fast pass |
| `ms-marco-deberta-v3-base` | 400MB | ~50ms | Muito boa | Accurate pass |
| Vertex AI Ranking API | Cloud | ~100ms | Excelente | Fallback (creditos) |

**Recomendacao**: Comecar apenas com `ms-marco-MiniLM-L-6-v2`. Adicionar deberta se qualidade insuficiente.

### Componentes

| Diretorio | Linguagem | Responsabilidade |
|-----------|-----------|------------------|
| `src/reranker/` | Python | FastAPI server, hybrid engine, model management |
| `src/training/` | Python | Fine-tuning com GCP, export ONNX + quantizacao |
| `src/lib/` | Rust | FFI scoring de alta performance, IPFS pinning |
| `modules/` | Nix | Declarative infra: service, cache, IPFS, monitoring |
| `configs/` | TOML/YAML | Registry de modelos, config GCP training |

### Decisoes de Design

- **Rust FFI (`scorer.rs`)**: scoring em batch via Rust para latencia sub-ms em comparacoes de similaridade
- **IPFS (`ipfs_client.rs`, `ipfs-cluster.nix`)**: modelos distribuidos e pinados, cache de embeddings persistente e content-addressed
- **Training pipeline**: fine-tuning automatizado no GCP (aproveita creditos), export para ONNX + quantizacao para deploy local
- **Nix modules**: cada componente e um modulo Nix independente, composavel no flake.nix
