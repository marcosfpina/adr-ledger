## Context

PHANTOM (v2.0.0, CORTEX-UNIFIED) é um document intelligence framework que realiza:

1. **Semantic Chunking**: Divide documentos em chunks de 1024 tokens com overlap de 128
2. **LLM Classification**: Extrai themes, patterns, concepts, learnings via LLM
3. **Vector Embedding**: Gera embeddings para similarity search
4. **FAISS Indexing**: Indexa vectors para retrieval

### Problema: Cloud API Costs Insustentáveis

**Cenário Real**: Analisar codebase de 10k arquivos Python

**Cálculo de Custos (Cloud-Only)**:
```
Files: 10,000
Avg file size: 500 lines = ~1500 tokens
Total tokens: 10k × 1500 = 15M tokens

OpenAI GPT-4 Turbo:
  Input: 15M × $0.03/1K = $450
  Output: 15M × 0.2 (20% response) × $0.06/1K = $180
  Total: $630 para uma única análise

Anthropic Claude Opus:
  Input: 15M × $0.015/1K = $225
  Output: 15M × 0.2 × $0.075/1K = $225
  Total: $450

Iterações: 5 análises durante development
Total: $2250-$3150
```

**Problemas Adicionais**:
- **Rate Limits**: OpenAI tier 2 = 10k req/min → 10k files levam ≥1 minuto (single-threaded)
- **Data Privacy**: Código proprietário enviado para OpenAI servers (US)
- **Latency**: 200-500ms por request × 10k = 33-83 minutos total time
- **Vendor Lock-in**: API changes, pricing increases, deprecations

### Performance Requirements

| Task                | Volume               | Latency Target    | Cost Target       |
|---------------------|----------------------|-------------------|-------------------|
| Semantic Chunking   | 10k docs/hour        | <30ms/doc         | $0 (CPU-only)     |
| LLM Classification  | 1k docs/hour         | <2s/doc           | <$0.01/doc        |
| Vector Embedding    | 10k docs/hour        | <100ms/doc        | $0 (local GPU)    |
| FAISS Search        | 100k queries/hour    | <10ms/query       | $0 (RAM/VRAM)     |

**Cloud APIs Cannot Meet These Targets**:
- Cost: $0.63/doc (GPT-4) vs target $0.01/doc
- Latency: 200-500ms vs target <2s total (chunking + classification + embedding)
- Rate limits: 10k req/min vs target 1k docs/hour (17 req/min sustained)

## Decision

Adotamos **Local-First AI Strategy** com **Hybrid Cloud Fallback**.

### Hybrid Cloud Strategy

**Decision Matrix**: When to use local vs cloud?

```python
def select_provider(task: Task) -> LLMProvider:
    """Smart provider selection based on task characteristics."""

    # Priority 1: Use local if available and sufficient
    if llamacpp.is_available() and task.complexity < THRESHOLD_COMPLEX:
        # 95% of tasks: routine classification, summarization
        return LlamaCppProvider()

    # Priority 2: Use cloud for complex reasoning
    if task.complexity >= THRESHOLD_COMPLEX:
        # 5% of tasks: complex analysis, multi-step reasoning
        if budget.has_credits():
            return OpenAIProvider()  # Best quality
        else:
            return DeepSeekProvider()  # Budget fallback

    # Priority 3: Fallback if local unavailable
    if not llamacpp.is_available():
        log.warning("Local inference down, using cloud fallback")
        return DeepSeekProvider()  # Cheaper than OpenAI
```

**Complexity Heuristics**:
- **Simple** (local): Extract keywords, summarize <500 tokens, classify into known categories
- **Complex** (cloud): Multi-document reasoning, creative generation, ambiguous edge cases

**PHANTOM Integration**:
```python
# src/phantom/providers/llamacpp.py
class LlamaCppProvider(LLMProvider):
    def __init__(self, base_url="http://localhost:8080"):
        self.client = openai.Client(base_url=base_url, api_key="dummy")

    def infer(self, prompt: str, max_tokens: int = 512) -> str:
        response = self.client.chat.completions.create(
            model="llama-3-8b-instruct",
            messages=[{"role": "user", "content": prompt}],
            max_tokens=max_tokens,
            temperature=0.7
        )
        return response.choices[0].message.content

    def estimate_cost(self, tokens: int) -> float:
        # Electricity: RTX 4090 = 450W, $0.12/kWh
        # 25 tokens/sec → 3600 tokens/minute
        # Cost: (450W / 1000) * $0.12 * (tokens / 3600 / 60)
        return (0.45 * 0.12 * tokens) / (3600 * 60)  # ~$0.0000015/token
```

### FAISS Vector Store

**Index Strategy**:
```python
# src/phantom/rag/vectors.py
class FAISSVectorStore:
    def __init__(self, dimension: int = 384):
        # Small scale (<10k vectors): Exact search
        if expected_size < 10_000:
            self.index = faiss.IndexFlatL2(dimension)

        # Medium scale (10k-1M vectors): Inverted index
        elif expected_size < 1_000_000:
            quantizer = faiss.IndexFlatL2(dimension)
            self.index = faiss.IndexIVFFlat(quantizer, dimension, nlist=100)

        # Large scale (>1M vectors): HNSW graph
        else:
            self.index = faiss.IndexHNSWFlat(dimension, M=32)

    def search(self, query_vector: np.ndarray, top_k: int = 5):
        distances, indices = self.index.search(query_vector, top_k)
        return [
            SearchResult(
                document_id=self.id_map[idx],
                distance=dist,
                metadata=self.metadata[idx]
            )
            for idx, dist in zip(indices[0], distances[0])
        ]
```

**Performance** (RTX 4090):
- Indexing: 60k vectors/min
- Search: <10ms for 100k vectors (exact)
- Storage: ~1.5KB per vector (384 dims × 4 bytes)

## Rationale

### Cost Analysis: Local vs Cloud (10k Documents)

| Component           | Local (llama.cpp + FAISS) | Cloud (OpenAI + Pinecone) |
|---------------------|---------------------------|---------------------------|
| **LLM Inference**   | $0.015 (electricity)      | $450 (GPT-4 Turbo)        |
| **Embeddings**      | $0 (local sentence-transformers) | $30 (text-embedding-3-large) |
| **Vector Storage**  | $0 (local FAISS)          | $70/month baseline        |
| **Search Queries**  | $0 (local)                | $0.40/million vectors     |
| **Total (one-time)**| **$0.015**                | **$480**                  |
| **Total (monthly)** | **$0.015**                | **$550+**                 |

**ROI**: Local setup (RTX 4090 = $1600) pays for itself after 3 uses.

### Privacy Analysis

**Data Flow (Cloud)**:
```
Proprietary code → Network → OpenAI servers (US) → Training data?
```

**Risks**:
- Code leakage to competitors (if OpenAI compromised)
- LGPD violation (data transfer outside Brazil without consent)
- SOC2 compliance risk (third-party data processing)

**Data Flow (Local)**:
```
Proprietary code → Local RAM/VRAM → Inference → Result → Local storage
```

**Benefits**:
- Zero data exfiltration
- LGPD compliant (data never leaves infrastructure)
- SOC2 compliant (full control over processing)
- Audit trail (all processing local, logged)

### Latency Comparison

**Cloud Workflow** (per document):
```
1. Chunk document: 10ms (CPU)
2. Send to OpenAI API: 50ms (network)
3. OpenAI inference: 200ms (queue + GPU)
4. Receive response: 50ms (network)
5. Parse response: 5ms (CPU)
Total: 315ms
```

**Local Workflow** (per document):
```
1. Chunk document: 10ms (CPU)
2. Send to llama.cpp (localhost): 1ms (loopback)
3. llama.cpp inference: 40ms (GPU)
4. Receive response: 1ms (loopback)
5. Parse response: 5ms (CPU)
Total: 57ms
```

**Speedup**: 5.5× faster local

# src/phantom/tools/vram_calculator.py
def check_vram_available() -> int:
    """Returns available VRAM in MB."""
    import torch
    if torch.cuda.is_available():
        free_vram = torch.cuda.mem_get_info()[0] // (1024 ** 2)
        return free_vram
    return 0

# Auto-throttling
if check_vram_available() < VRAM_WARNING_MB:
    log.warning(f"VRAM low: {vram_mb}MB, throttling workers")
    workers = max(1, workers // 2)

### Hybrid Fallback Example

**Scenario**: Complex multi-document reasoning query

```python
# User query: "Compare authentication patterns across 5 microservices"
query = "Compare auth in service-a, service-b, ..."

# PHANTOM attempts local
try:
    result = llamacpp_provider.infer(query, max_tokens=2048)
    confidence = estimate_confidence(result)

    if confidence < 0.7:
        # Low confidence, escalate to cloud
        log.info("Local result low-confidence, using GPT-4 fallback")
        result = openai_provider.infer(query, max_tokens=2048)
        cost = openai_provider.estimate_cost(2048)
        log.info(f"Cloud fallback cost: ${cost:.2f}")
except VRAMExhaustedError:
    # VRAM issue, use cloud
    log.warning("VRAM exhausted, cloud fallback")
    result = deepseek_provider.infer(query)
```

### Trade-offs Accepted

1. **Quality**: Llama 3 8B ~68% MMLU vs GPT-4 ~86%
   - **Acceptable**: Document classification, summarization (routine tasks)
   - **Mitigation**: Cloud fallback for complex queries

2. **Setup Complexity**: llama.cpp server, model downloads, VRAM tuning
   - **Cost**: ~1 week initial setup
   - **Benefit**: $4000+ savings per 10k doc analysis

3. **GPU Dependency**: Requires RTX 4090 or equivalent
   - **Cost**: $1600 upfront
   - **ROI**: Pays for itself after 3-5 analyses vs GPT-4
